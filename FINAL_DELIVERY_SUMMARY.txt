4. VALIDATIONS & SÉCURITÉ
═══════════════════════════════════════════════════════════════════════════════

VALIDATIONS STRICTES:

Classroom:
  ✅ name: Requis, unique, string, max 100
  ✅ specialty: Requis, string, max 100
  ✅ level: Requis, in[Seconde, Première, Terminale]

Student:
  ✅ matricule: Requis, unique, format regex
  ✅ birth_date: Requis, date, antérieur aujourd'hui
  ✅ classroom_id: Requis, exists:classrooms.id
  ✅ Status: active, transferred, expelled

Grade:
  ✅ value: Numérique, min:0, max:20 (STRICT)
  ✅ sequence: 1-6 uniquement
  ✅ Enseignant affecté à classe
  ✅ Élève actif uniquement
  ✅ Déduplication offline (client_uuid)

Absence:
  ✅ hours: Integer, min:1, max:8
  ✅ date: Date, before_or_equal:today
  ✅ Déduplication: 1/élève/jour/classe
  ✅ Normalisation dates

Remark:
  ✅ type: in[comportement, travail, assiduité]
  ✅ content: String, max:500
  ✅ Élève actif uniquement

CourseProgression:

8. GUIDE DE DÉPLOIEMENT & UTILISATION
═══════════════════════════════════════════════════════════════════════════════

INSTALLATION:

# 1. Clone & setup
git clone https://github.com/nexatec/edusmart-cm.git
cd edusmart-cm
composer install
cp .env.example .env
php artisan key:generate

# 2. Base de données
php artisan migrate
php artisan db:seed

# 3. Démarrer serveur
php artisan serve
# ➜ http://localhost:8000

VÉRIFICATION:

# Health check
curl http://localhost:8000/api/v1/health

# Lancer tous les tests
php artisan test --testdox

# Tests spécifiques
php artisan test tests/Feature/Admin/ --testdox
php artisan test tests/Feature/Teacher/ --testdox

UTILISATION - EXEMPLE COMPLET:

# 1. Créer classe
curl -X POST http://localhost:8000/api/v1/admin/classrooms \
  -H "Content-Type: application/json" \
  -d '{"name": "Terminale C", "specialty": "Sciences", "level": "Terminale"}'

# 2. Créer enseignant
curl -X POST http://localhost:8000/api/v1/admin/teachers \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Marie",
    "last_name": "Essomba",
    "email": "marie@example.cm",
    "main_subject_id": 1
  }'

# 3. Inscrire élève
curl -X POST http://localhost:8000/api/v1/admin/students \
  -H "Content-Type: application/json" \
  -d '{
    "matricule": "ESM-2026-001",
    "first_name": "Jean",
    "last_name": "Mbarga",
    "birth_date": "2008-03-12",
    "classroom_id": 1
  }'

# 4. Saisir notes (enseignant)
curl -X POST http://localhost:8000/api/v1/teacher/grades/bulk \
  -H "Content-Type: application/json" \
  -d '{
    "teacher_id": 1,
    "classroom_id": 1,
    "subject_id": 1,
    "sequence_id": 1,
    "grades": [{"student_id": 1, "value": 16}]
  }'

# 5. Générer bulletin
curl http://localhost:8000/api/v1/admin/students/1/bulletin?sequence_id=1
