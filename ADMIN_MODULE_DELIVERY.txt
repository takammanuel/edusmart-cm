1. VUE D'ENSEMBLE
═══════════════════════════════════════════════════════════════════════════════

Le Module Administration d'EDUSMART-CM fournit une API REST complète pour 
gérer l'intégralité des opérations administratives des établissements scolaires 
camerounais :

✅ Gestion des Classes (Création, modification, suppression)
✅ Gestion des Élèves (Inscription, transfert, radiation)
✅ Gestion des Enseignants (Personnel, assignations classe/matière)
✅ Génération des Bulletins (JSON + PDF avec calculs de moyennes)
✅ Calculs des Moyennes (Système pondéré camerounais)
✅ Suivi des Absences (Heures, justification, motifs)

Stack: Laravel 11 | PostgreSQL | REST API v1 | PHPUnit Tests

2. ARCHITECTURE TECHNIQUE
───────────────────────────────────────────────────────────────────────────────

MODÈLES & RELATIONS :

  Classroom ─────┐
                 ├──→ Student (1→N)
                 ├──→ Teacher (N↔N) via classroom_teacher
                 └──→ Grade (via students)

  Teacher ───────┐
                 ├──→ Subject (main_subject_id)
                 ├──→ Classroom (N↔N)
                 ├──→ Grade (1→N)
                 └──→ Absence (1→N)

  Student ───────┐
                 ├──→ Classroom (N→1)
                 ├──→ Grade (1→N)
                 └──→ Absence (1→N)

  Grade ─────────┐
                 ├──→ Student (N→1)
                 ├──→ Teacher (N→1)
                 ├──→ Subject (N→1)
                 ├──→ Classroom (N→1)
                 └──→ Sequence (N→1)

  Subject ───────┐
                 └──→ Grade (1→N)

  Sequence ──────┐
                 └──→ Grade (1→N)

  Absence ───────┐
                 ├──→ Student (N→1)
                 └──→ Classroom (N→1)

4. MODÈLES DE DONNÉES
───────────────────────────────────────────────────────────────────────────────

Classroom
  id (PK)
  name VARCHAR(100)                   ex: "Terminale C"
  specialty VARCHAR(100)              ex: "Sciences"
  level VARCHAR(50)                   ex: "Terminale"
  created_at, updated_at

Student
  id (PK)
  matricule VARCHAR(50) UNIQUE        ex: "ESM-2026-001"
  first_name VARCHAR(100)
  last_name VARCHAR(100)
  birth_date DATE
  classroom_id FK → classrooms.id
  status ENUM(active, transferred, expelled)
  created_at, updated_at

Teacher
  id (PK)
  first_name VARCHAR(100)
  last_name VARCHAR(100)
  email VARCHAR(100) UNIQUE
  main_subject_id FK → subjects.id
  created_at, updated_at

Subject
  id (PK)
  name VARCHAR(100)                   ex: "Mathématiques"
  code VARCHAR(10) UNIQUE             ex: "MATH"
  coefficient INT                     ex: 4
  created_at, updated_at

Sequence
  id (PK)
  number INT (1-6)
  name VARCHAR(100)                   ex: "1ère séquence"
  school_year VARCHAR(20)             ex: "2025-2026"
  is_active BOOLEAN
  created_at, updated_at

Grade
  id (PK)
  student_id FK → students.id
  teacher_id FK → teachers.id
  subject_id FK → subjects.id
  classroom_id FK → classrooms.id
  sequence_id FK → sequences.id
  value DECIMAL(4,2)                  [0-20]
  created_at, updated_at

Absence
  id (PK)
  student_id FK → students.id
  classroom_id FK → classrooms.id
  date DATE
  hours INT                           [1-8]
  is_justified BOOLEAN
  reason VARCHAR(255) optional
  created_at, updated_at

classroom_teacher (Pivot)
  classroom_id FK
  teacher_id FK
  subject_id FK
  PRIMARY KEY(classroom_id, teacher_id, subject_id)

7. GUIDE D'INSTALLATION
───────────────────────────────────────────────────────────────────────────────

PRÉREQUIS:
  • PHP 8.2+
  • Composer
  • PostgreSQL 12+ (ou MySQL 8+)
  • Node.js 18+ (optionnel, pour frontend)

INSTALLATION STEP-BY-STEP:

# 1. Cloner le repository
$ git clone https://github.com/nexatec/edusmart-cm.git
$ cd edusmart-cm

# 2. Installer dépendances PHP
$ composer install

# 3. Setup environnement
$ cp .env.example .env
$ php artisan key:generate

# 4. Configurer BD dans .env
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=edusmart_cm
DB_USERNAME=postgres
DB_PASSWORD=votre_mot_de_passe

# 5. Créer BD
$ createdb edusmart_cm

# 6. Migrations
$ php artisan migrate

# 7. Seeders (données test)
$ php artisan db:seed DatabaseSeeder

# 8. Lancer serveur
$ php artisan serve
# ➜ http://localhost:8000

# 9. Tester l'API (health check)
$ curl http://localhost:8000/api/v1/health
# Response: {"success":true,"service":"backend","version":"1.0.0"}

# 10. Lancer tests
$ php artisan test tests/Feature/Admin/ --testdox
# Résultat: 34 tests passed ✅

8. EXEMPLE D'UTILISATION COMPLÈTE
───────────────────────────────────────────────────────────────────────────────

SCÉNARIO: Créer une classe, ajouter des élèves, des enseignants et générer 
          un bulletin pour un élève.

ÉTAPE 1: Créer une classe
─────────────────────────
$ curl -X POST http://localhost:8000/api/v1/admin/classrooms \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Terminale C",
    "specialty": "Sciences",
    "level": "Terminale"
  }'

→ Response:
{
  "success": true,
  "message": "Ressource créée avec succès",
  "data": {
    "id": 1,
    "name": "Terminale C",
    "specialty": "Sciences",
    "level": "Terminale",
    "students_count": 0
  }
}

ÉTAPE 2: Créer un enseignant
──────────────────────────
$ curl -X POST http://localhost:8000/api/v1/admin/teachers \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Marie",
    "last_name": "Essomba",
    "email": "marie.essomba@example.cm",
    "main_subject_id": 1
  }'

→ Response: (teacher_id = 1)

ÉTAPE 3: Assigner l'enseignant à la classe
────────────────────────────────────────
$ curl -X POST http://localhost:8000/api/v1/admin/teachers/1/assignments \
  -H "Content-Type: application/json" \
  -d '{
    "classroom_id": 1,
    "subject_id": 1
  }'

ÉTAPE 4: Inscrire des élèves
───────────────────────────
$ curl -X POST http://localhost:8000/api/v1/admin/students \
  -H "Content-Type: application/json" \
  -d '{
    "matricule": "ESM-2026-001",
    "first_name": "Jean",
    "last_name": "Mbarga",
    "birth_date": "2008-03-12",
    "classroom_id": 1
  }'

→ Response: (student_id = 1)

ÉTAPE 5: Saisir des notes (via Module Enseignant)
─────────────────────────────────────────────────
$ curl -X POST http://localhost:8000/api/v1/teacher/grades/bulk \
  -H "Content-Type: application/json" \
  -d '{
    "teacher_id": 1,
    "classroom_id": 1,
    "sequence_id": 1,
    "grades": [
      {"student_id": 1, "subject_id": 1, "value": 16}
    ]
  }'

ÉTAPE 6: Générer un bulletin
───────────────────────────
$ curl http://localhost:8000/api/v1/admin/students/1/bulletin?sequence_id=1

→ Response:

      {
        "subject_name": "Mathématiques",

        "coefficient": 4,
        "grade": 16,

      "general_average": 16,
